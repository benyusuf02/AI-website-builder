Hoşgeldiniz

Bu rehber size ne satın alacağınızı, satın aldıklarınızı nasıl kuracağınızı (API Key alma vb.) ve kodları sunucuya nasıl atacağınızı adım adım anlatır.

1Alışveriş Listesi

Projeyi çalıştırmak için 3 şeye ihtiyacınız var. Bunları satın alın:

1. Sunucu (VPS)

Contabo önerilir. "VPS S" veya "VPS M" paketi.

Gerekli: Ubuntu 24.04
2. Alan Adı (Domain)

Namecheap önerilir. API desteği iyidir.

Örn: siteadi.com
3. AWS Hesabı

E-posta (SES) ve Dosya (S3) için Amazon AWS hesabı açın.

Kredi Kartı Gerekir
2. Servis Kurulumları (API Key Alma)

Satın aldınız ama bunları projeye nasıl bağlayacaksınız? "API Key" alarak. İşte yöntemleri:

  1. AWS Konsoluna giriş yapın ve arama çubuğuna IAM yazın.
  2. Soldan Users -> Create user deyin. Adına "YDeveloper" diyebilirsiniz.
  3. İzinler (Permissions) ekranında Attach policies directly seçin.
  4. Arama kutusuna yazıp şunları tikleyin: AmazonS3FullAccess ve AmazonSESFullAccess.
  5. Kullanıcıyı oluşturun. Sonra kullanıcıya tıklayın -> Security credentials sekmesine gelin.
  6. Create access key butonuna basın.
  7. Size Access Key ID (AKIA...) ve Secret Access Key verecek. BUNLARI KAYDEDİN! Bir daha göremezsiniz.
E-posta Onayı (SES):
  1. AWS aramaya Amazon SES yazın.
  2. Soldan Identities -> Create identity.
  3. Email address seçin, e-postanızı yazın ve Create deyin.
  4. E-postanıza gelen linke tıklayıp onaylayın. Artık bu mailden gönderim yapabilirsiniz.

Not: Namecheap API'yi açmak için hesabınızda en az 50$ bakiye olmalı veya 20 domain olmalı. Yoksa Sandbox (Test) kullanın.

  2. API Access'i Enable (Aktif) yapın.
  3. Whitelisted IPs kısmına Edit deyin ve Contabo'dan aldığınız Sunucu IP adresini ekleyin.
  4. Size verilen API Key'i kaydedin. ApiUser ise kullanıcı adınızdır.

  1. Iyzico Sandbox (Test) veya Gerçek panele girin.
  2. Sol altta Ayarlar menüsüne tıklayın.
  3. Üstteki sekmelerden Geliştirici (Developer) sekmesine geçin.
  4. Orada API Key ve Secret Key göreceksiniz. Kopyalayın.

3Projeyi Paketle

Kodları sunucuya göndermeden önce paketlememiz (Publish) lazım.

  1. Visual Studio'da projenize (Solution değil, proje ismine) sağ tıklayın.
  2. Publish seçeneğine tıklayın.
  3. Hedef (Target) olarak Folder (Klasör) seçin. Finish deyin.
  4. Publish butonuna basın.
  5. Çıktı klasöründeki dosya yolunu açın. Bu klasördeki her şeyi masaüstünde SiteDosyalari diye bir klasöre kopyalayın.
4Sunucu Kurulumu

Contabo'dan aldığınız IP ve şifre ile sisteme girin (PowerShell: ssh root@IPSUNUCU).

Sırayla şu komutları yapıştırın:

1. Temel Güncelleme:
sudo apt update && sudo apt upgrade -y
2. .NET 8 Kurulumu:
sudo apt-get install -y aspnetcore-runtime-8.0
3. Caddy (SSL) Kurulumu:
sudo apt install -y debian-keyring debian-archive-keyring apt-transport-https curl curl -1sLf 'https://dl.cloudsmith.io/public/caddy/stable/gpg.key' | sudo gpg --dearmor -o /usr/share/keyrings/caddy-stable-archive-keyring.gpg curl -1sLf 'https://dl.cloudsmith.io/public/caddy/stable/debian.deb.txt' | sudo tee /etc/apt/sources.list.d/caddy-stable.list sudo apt update sudo apt install caddy
5SQL Server Kurulumu (Docker)

Veritabanı sunucusu kurmak için en kolay yol Docker'dır.

1. Docker Kur:
sudo apt install -y docker.io && sudo systemctl start docker && sudo systemctl enable docker
2. SQL Server Başlat:

Aşağıdaki 'Sifreniz123!' kısmını değiştirin ve unutmayın!

sudo docker run -e "ACCEPT_EULA=Y" -e "SA_PASSWORD=Sifreniz123!" -p 1433:1433 --name sql_server --restart always -d mcr.microsoft.com/mssql/server:2022-latest
6Dosyaları Yükle
  1. FileZilla programını açın.
  2. Sunucu bilgilerinizi (IP, root, şifre, port 22) girip bağlanın.
  3. Sağ tarafta /var/www/ klasörüne gidin.
  4. Sağ tıklayıp sitem (veya istediğiniz bir isim) klasörü oluşturun.
  5. Masaüstündeki SiteDosyalari içindekileri buraya sürükleyin.
7. Ayarlar (En Önemli Kısım)

FileZilla'da appsettings.json dosyasına sağ tıklayıp düzenleyin. Adım 2'de aldığınız Key'leri buraya yapıştırın.

A. Veritabanı
"ConnectionStrings": {
  "YDeveloperContext": "Server=127.0.0.1,1433;Database=YDeveloperDB;User Id=sa;Password=Sifreniz123!;TrustServerCertificate=True;"
}
B. Diğer Servisler
"AWS": { "AccessKey": "AKIA...", "SecretKey": "..." },
"DomainRegistrar": { "ApiUser": "...", "ApiKey": "..." },
"Payment": { "ApiKey": "...", "SecretKey": "..." }
Veritabanını Kur: Ayarları kaydettikten sonra terminalde şu komutu çalıştırın:
cd /var/www/sitem && dotnet YDeveloper.dll --migrate
8Final: Çalıştır
1. Servis Dosyası Oluştur:
sudo nano /etc/systemd/system/ydeveloper.service
2. İçine Yapıştır (Klasör adını kendinize göre düzenleyin): 
[Unit]
Description=SaaS App
After=network.target

[Service]
WorkingDirectory=/var/www/sitem
ExecStart=/usr/bin/dotnet /var/www/sitem/YDeveloper.dll
Restart=always
RestartSec=10
KillSignal=SIGINT
SyslogIdentifier=ydeveloper-app
User=root
Environment=ASPNETCORE_ENVIRONMENT=Production

[Install]
WantedBy=multi-user.target

Kaydet: CTRL+X -> Y -> Enter.

3. Başlat:
sudo systemctl enable ydeveloper.service sudo systemctl start ydeveloper.service sudo cp /var/www/sitem/Caddyfile /etc/caddy/Caddyfile sudo systemctl reload caddy
